# ماموریت: بازیابی وضعیت پروژه — تمرکز روی Backend

تو الان به این پروژه دسترسی داری اما هیچ حافظه‌ای از مکالمات قبلی من با تو نداری.
فرض کن هیچ چیزی درباره این پروژه نمی‌دونی و باید همه چیز رو از خود کد کشف کنی.

⚠️ محدوده کار: فقط لایه بک‌اند (API، منطق سرور، دیتابیس، احراز هویت، سرویس‌ها، صف‌ها، زیرساخت).
به کامپوننت‌های UI و استایل کاری نداشته باش، مگر برای فهمیدن اینکه فرانت‌اند دقیقاً چه دیتایی از API انتظار داره.

قبل از پیشنهاد دادن یا نوشتن حتی یک خط کد جدید، مراحل زیر رو دقیقاً انجام بده:

## مرحله ۱ — شناخت معماری بک‌اند
- ساختار پوشه بک‌اند رو مرور کن (routes/, controllers/, models/, services/, middlewares/، و غیره).
- فریمورک (Express/NestJS/Django/FastAPI/...) و ORM/دیتابیس مورد استفاده رو شناسایی کن.
- فایل‌های .env.example، config/, و مستندات API (مثل Swagger/OpenAPI) رو بخون.
- schema دیتابیس یا migration ها رو بررسی کن تا مدل داده فعلی رو بفهمی.

## مرحله ۲ — تاریخچه کار روی بک‌اند
- `git log --oneline -30` رو بررسی کن و commit های مربوط به routes/models/services رو جدا کن.
- `git status` و `git diff` رو چک کن، مخصوصاً روی فایل‌های سرور، migration، و config.
- migration های اجراشده و اجرانشده رو مقایسه کن؛ ببین دیتابیس با آخرین schema هماهنگه یا نه.
- دنبال کامنت‌های `// TODO`, `// FIXME`, endpoint هایی که پیاده‌سازی نشدن (مثلاً `res.send("not implemented")`) بگرد.

## مرحله ۳ — سلامت فعلی بک‌اند
- سرور رو (اگه امکانش هست) اجرا کن و ببین بالا میاد یا خطا می‌ده (مخصوصاً اتصال دیتابیس).
- تست‌های backend (unit/integration) رو اجرا کن و نتیجه رو گزارش کن.
- endpoint های موجود رو لیست کن و مشخص کن کدوم واقعاً کار می‌کنن، کدوم نیمه‌کاره‌ن، کدوم فقط تعریف شدن ولی منطق ندارن.
- بررسی کن آیا مسائل امنیتی واضح (مثل کلید هاردکد شده، عدم اعتبارسنجی ورودی) وجود داره.

## مرحله ۴ — گزارش قبل از هر اقدام
این گزارش رو دقیقاً با همین ساختار بده:

**۱. خلاصه معماری بک‌اند:** استک فنی و ساختار کلی (۲-۳ جمله).
**۲. مدل داده فعلی:** خلاصه‌ای از schema/جدول‌ها و روابط اصلی.
**۳. آخرین کار انجام‌شده:** بر اساس commit ها و diff.
**۴. کارهای نیمه‌کاره:** endpoint های ناقص، migration های اجرانشده، تست‌های شکسته.
**۵. فرضیه من درباره نقطه توقف:** با ذکر شواهد مشخص.
**۶. سوالات من از تو:** هرچیزی که فقط با دسترسی به تصمیمات قبلی من قابل تأیید هست.
**۷. پیشنهاد قدم بعدی:** ۱-۲ گزینه مشخص، فقط در محدوده بک‌اند.

⚠️ تا وقتی این گزارش رو ننوشتی و من تأییدش نکردم، هیچ کد جدیدی ننویس و migration جدیدی اجرا نکن.
اگه چیزی درباره تصمیمات معماری قبلی مشخص نبود، حدس نزن — بپرس.
