# ماموریت: بازیابی وضعیت پروژه — تمرکز روی UI/UX

تو الان به این پروژه دسترسی داری اما هیچ حافظه‌ای از مکالمات قبلی من با تو نداری.
فرض کن هیچ چیزی درباره این پروژه نمی‌دونی و باید همه چیز رو از خود کد کشف کنی.

⚠️ محدوده کار: فقط لایه UI/UX (کامپوننت‌ها، صفحات، استایل‌ها، تجربه کاربری، دسترسی‌پذیری).
به منطق بک‌اند، API، دیتابیس یا زیرساخت سرور دست نزن و درباره‌شون هم نظر نده مگر اینکه مستقیماً روی UI اثر بذاره
(مثلاً یک state که باعث باگ بصری می‌شه).

قبل از پیشنهاد دادن یا نوشتن حتی یک خط کد جدید، مراحل زیر رو دقیقاً انجام بده:

## مرحله ۱ — شناخت لایه فرانت‌اند
- ساختار پوشه فرانت‌اند رو مرور کن (components/, pages/, styles/, hooks/, و غیره).
- فریمورک UI (React/Vue/...)، کتابخانه استایل (Tailwind/CSS Modules/Styled Components/...) و سیستم کامپوننت (مثل shadcn، MUI) رو شناسایی کن.
- اگه design system یا فایل توکن‌های طراحی (رنگ، فونت، فاصله‌گذاری) وجود داره، پیدا کن و بخون.
- هر فایل Storybook، Figma link، یا مستندات UI رو بررسی کن.

## مرحله ۲ — تاریخچه کار روی UI
- `git log --oneline -30` رو بررسی کن و فقط commit هایی که به فایل‌های UI/استایل مربوطن رو جدا کن.
- `git status` و `git diff` رو چک کن، مخصوصاً روی فایل‌های .css/.tsx/.jsx/.vue.
- دنبال کامنت‌های `// TODO`, `// FIXME` توی کامپوننت‌ها بگرد.
- ببین آیا کامپوننتی نصفه‌کاره مونده (مثلاً prop تعریف‌شده ولی استفاده‌نشده، state بلااستفاده، JSX ناقص).

## مرحله ۳ — سلامت فعلی UI
- پروژه رو (اگه امکانش هست) اجرا کن و صفحات اصلی رو چک کن که رندر می‌شن یا خطا می‌دن.
- به دنبال اینکونسیستنسی بصری بگرد: کامپوننت‌های تکراری با استایل متفاوت، فونت/رنگ‌های خارج از سیستم طراحی.
- وضعیت ریسپانسیو بودن (موبایل/دسکتاپ) و رعایت حداقل‌های دسترسی‌پذیری (کنتراست، فوکوس کیبورد، آلت‌تکست) رو بررسی کن.
- تست‌های مرتبط با UI (اگه هست، مثل Testing Library یا Cypress) رو اجرا کن.

## مرحله ۴ — گزارش قبل از هر اقدام
این گزارش رو دقیقاً با همین ساختار بده:

**۱. خلاصه بخش UI پروژه:** این محصول از نظر واسط کاربری چیه (۲-۳ جمله).
**۲. سیستم طراحی فعلی:** فریمورک، کتابخانه استایل، و اینکه چقدر منسجم رعایت شده.
**۳. آخرین کار UI انجام‌شده:** بر اساس commit ها و diff.
**۴. کارهای نیمه‌کاره UI:** کامپوننت ناقص، استایل ناهماهنگ، صفحات نیمه‌ساز.
**۵. فرضیه من درباره نقطه توقف:** با ذکر شواهد.
**۶. سوالات من از تو:** هرچیزی که نمی‌شه از کد فهمید (مثلاً تصمیم طراحی عمدی بوده یا باگ).
**۷. پیشنهاد قدم بعدی:** ۱-۲ گزینه مشخص، فقط در محدوده UI/UX.

⚠️ تا وقتی این گزارش رو ننوشتی و من تأییدش نکردم، هیچ کد جدیدی ننویس.
به فایل‌های بک‌اند/API دست نزن مگر برای فهمیدن اینکه یک داده از کجا میاد (فقط خواندن، نه ویرایش).
