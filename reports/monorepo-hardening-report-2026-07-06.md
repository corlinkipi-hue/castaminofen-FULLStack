# گزارش نهایی سخت‌سازی و اعتبارسنجی monorepo

تاریخ: 2026-07-06

## 1. خلاصه اجرا

در این بازه، ساختار monorepo برای پروژه Castaminofen به‌روزرسانی و تثبیت شد تا workflowهای اصلی شامل generate، build، typecheck، lint و test از یک مسیر قابل اعتماد و قابل تکرار اجرا شوند.

## 2. تغییرات اصلی انجام‌شده

- بازنویسی و سازمان‌دهی مجدد اسکریپت‌های ریشه برای وضوح بیشتر در وظایف هر مرحله.
- تفکیک مسئولیت‌های Turbo برای جلوگیری از در هم آمیختن generate و build.
- بهبود تنظیمات TypeScript برای پشتیبانی بهتر از Workspaceها و path mapping.
- جدا کردن Prisma generation از build و آماده‌سازی پیکربندی مستقل برای آن.
- هماهنگ‌سازی تنظیمات web، mobile و api-gateway با الگوی یکسان برای lint و typecheck.
- تنظیم lint برای برنامه موبایل با استفاده از ESLint flat config و حذف وابستگی به مسیرهای قدیمی Expo که با نسخه نصب‌شده سازگار نبود.

## 3. وضعیت تاییدشده

اعتبارسنجی‌های زیر با خروجی موفق انجام شدند:

- pnpm lint
- pnpm typecheck
- pnpm build

## 4. نتایج حاصل

- lint برای کل monorepo با موفقیت اجرا شد.
- typecheck برای همه بسته‌ها و برنامه‌ها بدون خطا انجام شد.
- build برای وب و موبایل با موفقیت تکمیل شد.
- مسیر mobile lint از حالت تعلیق و ناسازگار به مسیر اجرایی و قابل استفاده در CI تبدیل شد.

## 5. موارد باقیمانده

- چند warning lint در برنامه موبایل باقی مانده‌اند، اما هیچ خطای阻کننده‌ای برای build یا lint نهایی وجود ندارد.
- برای بهینه‌سازی نهایی، می‌توان در مرحله بعدی روی حذف warningها و استانداردسازی بیشتر کد تمرکز کرد.

## 6. پیشنهادهای بعدی

1. حذف warningهای باقیمانده در mobile برای تمیزتر شدن pipeline.
2. اضافه‌کردن CI workflow برای اجرای خودکار lint/typecheck/build در pull request.
3. گسترش تست‌های خودکار برای web، mobile و api-gateway.
4. یکپارچه‌سازی بیشتر design tokens و shared logic بین وب و موبایل.
