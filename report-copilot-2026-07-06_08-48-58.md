# ۱. معرفی کوتاه پروژه

پروژه Castaminofen یک وب اپ رسانه‌ای فارسی است که هدف آن ارائه تجربه مصرف پادکست، کتاب صوتی و ویدیو در یک تجربه یکپارچه است. این وب اپ با رویکرد mobile-first، RTL و dark mode پیش‌فرض طراحی شده و سعی دارد رابطی ساده، روان و مناسب بازار فارسی ارائه دهد.

وضعیت فعلی پروژه: UI صفحات و کامپوننت‌های اصلی پیاده شده ولی بیشتر آنها به صورت اولیه و بدون پیاده‌سازی کامل backend هستند. به عبارت دیگر، این پروژه در مرحله طراحی و UI وحود دارد و پیاده‌سازی عملکرد کامل آن هنوز نهایی نشده است.

---

# ۲. خلاصه وضعیت رابط کاربری

- کیفیت کلی UI: طراحی فعلی مناسب یک MVP UI است و اصول اولیه برند و تجربه فارسی را رعایت می‌کند.
- میزان پیشرفت: وب اپ دارای صفحات اصلی و رابط کاربری قابل مشاهده است. بسیاری از جریان‌ها فعلاً در لایه UI پیاده شده‌اند اما عملکرد کامل backend هنوز متصل نشده است.
- نقاط قوت:
  - سیستم رنگ و تم تاریک هماهنگ با مستندات و توکن‌های UI.
  - RTL کامل برای وب و فونت فارسی Vazirmatn.
  - ناوبری منطقی با لینک‌های اصلی مثل کاوش، جستجو، کتابخانه، ورود و ثبت‌نام.
  - componentهای پایه مثل ContentCard، PlayerBar، EmptyState، PaywallModal و CreatorContentCard آماده هستند.
- نقاط ضعف:
  - برخی بخش‌ها فقط UI هستند و عملکرد نهایی هنوز تکمیل نشده.
  - design tokens بین وب و موبایل هنوز به صورت نرم‌افزاری یکپارچه نشده‌اند.
  - صفحه‌های موبایل مانند Downloads و برخی ابزارها placeholder دارند.
  - تجربه پرداخت و خرید در UI وجود دارد اما بخش backend آن هنوز به صورت واقعی متصل نیست.

# ۳. معماری Frontend

- Framework: وب با `Next.js 15` پیاده شده است.
- زبان برنامه‌نویسی: `TypeScript`.
- Build Tool: `pnpm` و `turbo` در monorepo.
- Routing: `Next.js App Router` برای وب.
- State Management: `zustand` برای مدیریت state در وب و از آن مشابه در موبایل استفاده شده است.
- Styling System: وب با CSS variables و global CSS. mobile از inline styles/StyleSheet استفاده می‌کند.
- Component Library: کامپوننت‌های اختصاصی داخلی. کتابخانه UI شخص ثالث بزرگ مشاهده نشد.
- Animation Library: نشانه‌ای از کتابخانه انیمیشن خارجی وجود ندارد؛ انیمیشن‌ها به صورت ساده و استاندارد انجام شده است.
- Theme System: `@castaminofen/ui-tokens` و `data-theme` در وب.
- Design Tokenها: `packages/ui-tokens/tokens.css` و `packages/ui-tokens/src/tokens.ts` توکن‌های رنگ و radius را تعریف کرده‌اند.
- معماری پوشه‌ها: monorepo با `apps/web`, `apps/mobile`, `packages/shared`, `packages/ui-tokens`, `services/api-gateway`.

# ۴. بررسی سیستم طراحی

- رنگ‌ها: پالت برند براساس بنفش، سبز و خاکستری تیره مشخص است و در CSS tokens تعیین شده است.
- Typography: فونت Vazirmatn در وب استفاده شده؛ تایپوگرافی فارسی روی متون اصلی رعایت شده است.
- Spacing: در موبایل spacing tokenها وجود دارند، در وب اکثر فاصله‌ها با کلاس‌ها و متغیرهای CSS تعیین شده است.
- Componentها: مجموعه کامپوننت‌های پایه وجود دارد اما یک سیستم طراحی متمرکز و کتابخانه‌ای واحد هنوز کامل نشده است.
- Layoutها: همه صفحه‌های اصلی در وب و موبایل ساختار قابل قبول دارند. در وب layout صفحه با sticky nav و grid محتوا درست شده و موبایل با bottom tabs طراحی شده است.
- Consistency: هماهنگی کلی وجود دارد اما نیاز به یکپارچه‌سازی بیشتر بین وب و موبایل و نامگذاری یکنواخت دارد.

نکات قابل توجه:
- توکن‌ها برای رنگ و شعاع گوشه‌ها تعریف شده‌اند ولی هنوز یک استراتژی قوی برای تایپوگرافی، spacing و component-level tokens وجود ندارد.
- بعضی از کامپوننت‌ها مانند `btn-primary`, `form-input` و `content-card` به صورت تک‌به‌تک و جدا طراحی شده‌اند که می‌تواند بعدها یکپارچه شود.

# ۷. بخش‌های نیمه‌کاره

- `apps/mobile/app/(tabs)/downloads.tsx`: صفحه دانلودها صرفاً placeholder فاز 2 است.
- `apps/mobile/app/(tabs)/tools.tsx`: بخش‌های یادشده به‌زودی بوده و UI کامل نشده.
- `apps/mobile/app/video/[id].tsx`: بخش اپیزودهای بیشتر به‌زودی نمایش داده می‌شود.
- `apps/web/src/components/PaywallModal.tsx`: UI پرداخت وجود دارد ولی عملکرد پرداخت واقعی هنوز به backend متصل نشده است.
- `apps/web/src/components/EpisodeUploadForm.tsx`: فرم آپلود در UI موجود است اما اتصال کامل backend فعلاً مشخص نیست.
- `apps/mobile/app/content/[id].tsx`: content detail موبایل پایه دارد اما بعضی تجربه‌ها و حالت‌های premium نیاز به تکمیل دارند.
- `packages/ui-tokens`: توکن‌های طراحی موجود هستند اما همگرا کردن کامل آن‌ها بین وب و موبایل هنوز انجام نشده است.

# ۸. ابهام‌هایی که فقط صاحب پروژه می‌تواند پاسخ دهد

با پاسخ‌های جدید شما، این موارد اکنون روشن‌تر شده‌اند:
- `Downloads` و آفلاین در فاز 2 قرار دارند.
- Paywall و پرداخت در UI واقعی محسوب می‌شوند، ولی فعلاً فقط UI شده‌اند.
- workflow سازنده باید شامل طراحی صفحات کامل باشد.
- هدف نهایی، همگرایی design tokens بین وب و موبایل است.
- این پروژه یک وب اپ خواهد بود.
- در حال حاضر فقط لایه UI ساخته شده است و عملکرد کامل backend در بسیاری نقاط تکمیل نشده.

سوالات باقی‌مانده:
- آیا پردازش‌های واقعی پرداخت و verify قرار است در همین سرویس `api-gateway` پیاده شود یا بخشی از فاز بعدی خواهد بود؟
- آیا صفحه سازنده باید نهایتاً شامل ویرایش اپیزود، مدیریت تیم سازنده و داشبورد آنالیتیکس شود؟
- آیا یک کتابخانه کامپوننت مشترک بین وب و موبایل باید به صورت مجزا ساخته شود یا کافی است همان توکن‌ها مشترک باشند؟

# ۹. پیشنهاد قدم بعدی

1. تمرکز روی تکمیل UI `PaywallModal` و اتصال آن به backend واقعی پرداخت. این کار نشان می‌دهد که تجربه پریمیوم فقط UI نیست و تبدیل به یک جریان کاربری کامل شده است.
2. ایجاد یک لایه design token واحد برای وب و موبایل. این کار باعث می‌شود توسعه UI جدید سریع‌تر، هماهنگ‌تر و بدون تکرار انجام شود.
